# CachyOS Dotfiles
Custom dotfiles for my new CachyOS install.

## Usage

Stow the packages you want into `$HOME`:

```bash
stow -t ~ dolphin fish hypr kitty nvim quickshell wallust
```

## Requirements

These dotfiles assume an Arch/CachyOS-style system with GNU Stow and the packages below available.

### Core

- `stow` for symlinking the repo into `$HOME`
- `hyprland` as the running compositor
- `kitty` as the terminal
- `fish` as the interactive shell
- `neovim` as the editor

### Hyprland

Required by [`hyprland.conf`](/home/zeke/Documents/git/cachyos-dotfiles/hypr/.config/hypr/hyprland.conf):

- `hyprlock`
- `hyprpaper`
- `quickshell`
- `hyprpolkitagent` as a user service started via `systemctl --user start hyprpolkitagent`
- `dolphin`
- `hyprlauncher`
- `fuzzel`

Notes:

- Hyprland is configured to autostart `quickshell`.
- Hyprland is configured to autostart `hyprpaper` using [`hyprpaper.conf`](/home/zeke/Documents/git/cachyos-dotfiles/hypr/.config/hypr/hyprpaper.conf).
- The lock binding expects `hyprlock`.
- The shutdown binding prefers `hyprshutdown` if installed, otherwise falls back to `hyprctl dispatch exit`.
- `SUPER+SHIFT+W` opens a `fuzzel` picker over `$HOME/Dropbox/Wallpapers` and runs [`set-wallpaper.sh`](/home/zeke/Documents/git/cachyos-dotfiles/hypr/.config/hypr/set-wallpaper.sh), which updates `hyprpaper`, rewrites `~/.config/hypr/current-wallpaper`, and runs `wallust`.

### Quickshell

Required by the files in [`quickshell/.config/quickshell/default`](/home/zeke/Documents/git/cachyos-dotfiles/quickshell/.config/quickshell/default):

- `quickshell`
- `pipewire`
- `wireplumber`
- a running StatusNotifier/system tray environment for tray icons
- `papirus-icon-theme`
- `pavucontrol` for the launcher button in the volume panel

Notes:

- [`shell.qml`](/home/zeke/Documents/git/cachyos-dotfiles/quickshell/.config/quickshell/default/shell.qml) sets `//@ pragma IconTheme Papirus`.
- Workspace state depends on Quickshell's Hyprland integration, so this setup is tied to Hyprland.
- Audio widgets depend on PipeWire default sink/source being available.
- Some tray apps provide icons via custom search paths (`name?path=/...`). Quickshell currently ignores those custom paths, so a few tray icons may still be missing unless the selected icon theme already contains them.

### Wallust

Required by [`wallust.toml`](/home/zeke/Documents/git/cachyos-dotfiles/wallust/.config/wallust/wallust.toml) and [`Theme.qml`](/home/zeke/Documents/git/cachyos-dotfiles/quickshell/.config/quickshell/default/Theme.qml):

- `wallust`
- the `wallust` stow package must be linked to `~/.config/wallust`
- Wallust must generate [`~/.cache/wallust/colors.json`](/home/zeke/.cache/wallust/colors.json)
- Wallust should also generate [`~/.local/share/color-schemes/Wallust.colors`](/home/zeke/.local/share/color-schemes/Wallust.colors) for KDE/Dolphin

Notes:

- Quickshell watches `~/.cache/wallust/colors.json` and updates colors live when Wallust regenerates it.
- The template in [`templates/colors.json`](/home/zeke/Documents/git/cachyos-dotfiles/wallust/.config/wallust/templates/colors.json) depends on Wallust's `pywal = true` templating mode.
- The template in [`templates/Wallust.colors`](/home/zeke/Documents/git/cachyos-dotfiles/wallust/.config/wallust/templates/Wallust.colors) generates a KDE color scheme for Dolphin and other KDE apps.

### Dolphin / KDE Apps

Required by [`dolphin/.config/kdeglobals`](/home/zeke/Documents/git/cachyos-dotfiles/dolphin/.config/kdeglobals):

- `dolphin`
- `kconfig`
- `kcolorscheme`

Notes:

- [`kdeglobals`](/home/zeke/Documents/git/cachyos-dotfiles/dolphin/.config/kdeglobals) points KDE apps at the `Wallust` color scheme.
- The actual palette file is generated by Wallust into `~/.local/share/color-schemes/Wallust.colors`.
- Dolphin usually needs to be restarted after the scheme file changes to pick up a new palette.

### Kitty

Required by [`kitty.conf`](/home/zeke/Documents/git/cachyos-dotfiles/kitty/.config/kitty/kitty.conf):

- `kitty`

Notes:

- Kitty includes [`current-theme.conf`](/home/zeke/Documents/git/cachyos-dotfiles/kitty/.config/kitty/current-theme.conf) as its active theme file.
- If you want the terminal to follow wallpaper colors too, that currently needs a separate Wallust or theme-generation step; the repo does not wire Kitty to Wallust automatically.

### Fish

Required by [`config.fish`](/home/zeke/Documents/git/cachyos-dotfiles/fish/.config/fish/config.fish) and [`fnm.fish`](/home/zeke/Documents/git/cachyos-dotfiles/fish/.config/fish/conf.d/fnm.fish):

- `fish`
- `fnm`
- `/usr/share/cachyos-fish-config/cachyos-config.fish` from CachyOS's fish config package

Notes:

- The shell config assumes the CachyOS fish defaults are installed system-wide.
- `fnm env --use-on-cd --shell fish` runs at shell startup.

Plugins configured directly in `init.lua`:

- `lazy.nvim`
- `plenary.nvim`
- `telescope.nvim`
- `catppuccin`
- `tokyonight.nvim`
- `kanagawa.nvim`
- `nightfox.nvim`
- `nvim-tree.lua`
- `nvim-web-devicons`

Notes:

- [`lazy-lock.json`](/home/zeke/Documents/git/cachyos-dotfiles/nvim/.config/nvim/lazy-lock.json) is present, so plugin versions are pinned once installed.
